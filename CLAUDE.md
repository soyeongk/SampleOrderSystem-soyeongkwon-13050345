# CLAUDE.md

이 파일은 이 저장소에서 작업하는 Claude Code에게 프로젝트 컨텍스트와 구현 요구사항을 제공합니다.

## 프로젝트 개요

**반도체 시료 생산주문관리 시스템 (SampleOrderSystem)**

가상의 반도체 회사 "S-Semi"는 다양한 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체,
대학 연구실 등에 납품한다. 최근 주문량 급증으로 엑셀/메모장 기반 관리의 한계(주문 처리 누락, 완료
시점 파악 불가, 불필요한 중복 생산 등)가 드러나, 이를 대체할 콘솔 기반 주문·재고·생산 관리 시스템을
개발한다.

### 역할별 흐름
1. **고객**: 필요한 시료를 이메일 등으로 요청 (시료 요청자)
2. **주문 담당자**: 요청에 맞게 주문서 작성 및 관리, 생산 담당자에게 주문서 전달
3. **생산 담당자**: 개발 시료 등록, 주문 수신 후 승인/거절, 시료 생산·승인

### 시스템 특성
- **콘솔(CLI) 기반**: 담당자가 직접 명령을 입력하여 시료 등록 및 주문 처리
- **생산 라인**: 공장에서 시료 하나를 생산하는 설비 흐름. 하나의 생산 라인은 시료를 하나씩 생산하며,
  주문이 들어온 시료에 대해서만 생산 진행

## 도메인 모델

### 시료 (Sample)
시스템의 가장 기본 단위. 등록된 시료만 주문 가능.

| 속성 | 설명 |
|---|---|
| 시료 ID | 고유 식별자 |
| 이름 | 시료명 |
| 평균 생산시간 | 1개 생산에 걸리는 평균 시간 |
| 수율 | 정상적인 시료 수 / 총 생산 시료 수 (예: 100개 중 90개 정상 = 0.9) |
| 현재 재고 | 조회 시 표시 |

### 주문 (Order)
| 상태 | 의미 |
|---|---|
| RESERVED | 주문 접수 |
| REJECTED | 주문 거절 (정상 흐름 외 상태, **모니터링 대상에서 제외**) |
| PRODUCING | 주문 승인 완료 + 재고 부족으로 생산 중 |
| CONFIRMED | 주문 승인 완료 + 출고 대기 중 |
| RELEASED | 출고 완료 |

주문 상태 전이:
```
RESERVED --(거절)--> REJECTED
RESERVED --(승인, 재고 충분)--> CONFIRMED --(출고 처리)--> RELEASED
RESERVED --(승인, 재고 부족)--> PRODUCING --(생산 완료)--> CONFIRMED --(출고 처리)--> RELEASED
```

주문 예약 시 입력 값: 시료 ID, 고객명, 주문 수량

## 기능 명세

### 1. 메인 메뉴
- 기능별 선택 화면(시료 관리 / 시료 주문 / 주문 승인·거절 / 모니터링 / 생산 라인 조회 / 출고 처리) 표시
- 전체 시료에 대한 요약 정보(등록 시료 수, 총 재고, 전체 주문 수, 생산라인 대기 등) 표시
- 화면 구성은 자유롭게 결정 (예시 UI는 참고용)

### 2. 시료 관리
- **시료 등록**: 시료 ID, 이름, 평균 생산시간, 수율 입력받아 신규 등록
- **시료 조회**: 등록된 모든 시료 목록 + 현재 재고 수량 표시
- **시료 검색**: 이름 등 속성으로 특정 시료 검색

### 3. 시료 주문 (예약)
- 고객이 원하는 시료 ID, 고객명, 주문 수량을 입력받아 주문 생성
- 시료 ID는 S-000 형태
- 생성 시점의 주문 상태는 `RESERVED`
- 주문번호가 생성되어야 하며 ORD-YYYYMMDD-HHMM 로 저장
- 주문건 마다 현재 상태가 계속 업데이트 되어야 함

### 4. 주문 승인/거절
- `RESERVED` 상태의 주문 목록 표시
- **승인**:
  - 재고 충분 → 즉시 `CONFIRMED` 로 전환
  - 재고 부족 → 생산 라인(생산 큐)에 자동 등록, 상태를 `PRODUCING` 으로 전환
- **거절**: 즉시 `REJECTED` 로 전환

### 5. 모니터링
- **주문량 확인**: 상태별(RESERVED/CONFIRMED/PRODUCING/RELEASED) 주문 수 표시. `REJECTED`는 유효하지 않은 주문이므로 집계에서 제외
- **재고량 확인**: 시료별 현재 재고 수량 + 상태 표기
  - 여유: 주문 대비 재고 충분
  - 부족: 주문 대비 재고 수량 부족
  - 고갈: 재고 수량 0

### 6. 생산 라인
- 주문 수량의 부족분을 생산하되, 수율/오차를 고려하여 계산
  - **실 생산량** = `ceil(부족분 / 수율)`
  - **총 생산 시간** = `평균 생산시간 * 실 생산량`
- 생산 완료 시 주문 상태를 `PRODUCING → CONFIRMED` 로 전환
- 생산 도중에 주문 수량의 부족분을 생산했다고 바로 CONFIRMED가 되는 것이 아니라 총 생산 시간이 모두 끝나야 CONFIRMED로 변경될 수 있음 
- **생산 현황 표기**: 현재 생산 중인 시료 정보 표시 (표기 정보 수준은 자율 결정, 예: 주문 정보/현재까지 생산량)
- **대기 주문 확인**: 생산 큐(대기열)에 있는 대기 목록 출력
  - 스케줄링 전략: **FIFO** (선입선출)

### 7. 출고 처리
- `CONFIRMED` 상태 주문에 대해서만 출고 가능
- 특정 주문에 대해 출고 실행 → 상태를 `RELEASED` 로 전환

프로젝트 본 구현 전, 아래 4가지 항목을 개별 PoC로 검증한다. **각 PoC는 개별 저장소로 분리 제출.**

|---|---|

| 데이터 영속성 처리 | 선택한 방식(파일, JSON, DB 등)으로 데이터 저장·조회 구조 구현, CRUD 포함. Application 재실행 후에도 데이터 유지 |
